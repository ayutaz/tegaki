Tegaki animates Japanese handwriting with **correct MEXT stroke order** (via [KanjiVG](https://kanjivg.tagaini.net/)) and a **human-like non-uniform pen speed** (via the Plamondon Sigma-Lognormal rhythm model). The easiest entry point is the `tegaki/fonts/ja-kana` pre-built bundle, which covers all 180 hiragana and katakana codepoints out of the box.

## How stroke order is decided

For any character the pipeline matches `[\u3040-\u309F\u30A0-\u30FF\u4E00-\u9FFF]`, the Stage 4 skeletonizer is swapped for a KanjiVG lookup:

1. `getKanjiSvg(codepoint)` returns the raw SVG from the pinned release (currently `r20250816`).
2. `parseKanjiSvg(svg)` extracts the `<path>` elements in document order ‚ÄĒ that order **is** the MEXT stroke order. The `kvg:type` attribute („áź „áĎ „áí „áÄ „áĒ ‚Ä¶) maps each stroke to a `tome` / `hane` / `harai` / `dot` / `default` endpoint classification.
3. The 109-normalized coordinates are mapped into the font's `pathBBox` with aspect-preserving letterboxing, then pushed through `raster.transform` to land in bitmap pixels.
4. `orderStrokes()` preserves the KanjiVG order as-is (no heuristic override).
5. When `--rhythm lognormal` is set, each per-point `t` value is remapped through a Plamondon lognormal CDF whose ŌÉ/őľ are modulated by the stroke's endpoint type ‚ÄĒ `tome` decelerates sharply, `hane` flicks late, `harai` skews right, `dot` is a short near-symmetric pulse.

Uncovered characters (emoji, Latin letters inside a Japanese string, JIS Level 3/4 kanji) fall through to the existing geometric heuristic. Pass `--strict` to abort instead.

## Known limitations

- **Kanji coverage ‚Č† full JŇćyŇć yet.** The dataset package ships kana only by default (`TEGAKI_KANJIVG_FULL=1` pulls in the full CJK Unified range during a refresh). A proper JŇćyŇć+JinmeiyŇć allowlist + a pre-built `ja-full` bundle are tracked as a Phase 7+ follow-up.
- **KanjiVG has occasional stroke-order errors.** Reported issues in the common band include [Ś®© (#155)](https://github.com/KanjiVG/kanjivg/issues/155) and [Śļę„ÉĽÁā≠ (#25)](https://github.com/KanjiVG/kanjivg/issues/25). Tegaki provides an escape hatch ‚ÄĒ drop a corrected SVG keyed by uppercase hex into `packages/dataset-cjk-kanjivg/fix-overrides.json` and it takes precedence over the upstream file.
- **Vertical writing is out of scope.** The first release assumes horizontal left-to-right layout; `writing-mode: vertical-rl` is a future enhancement.
- **The Sigma-Lognormal defaults are calibrated on healthy adults.** Children, elderly writers, and dysgraphia-style variants are modeled as parameter shifts (larger ŌÉ) but not shipped as presets. Tune `LOGNORMAL_SIGMA_DEFAULT` / `LOGNORMAL_MU_DEFAULT` in `packages/generator/src/constants.ts` to explore.
